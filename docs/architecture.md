# JustLinkIt - アーキテクチャ及び仕様設計書

## 1. システム概要
Snipping Toolの自動保存ファイル（画像および動画）を検知し、バックエンドへ自動アップロードした上で、生成された公開URLをクリップボードにコピーする常駐型軽量ツール。さらに「送る」メニューやクリップボードからの手動アップロードにも対応。

## 3. 追加仕様・決定事項

### 3.1. クライアント側の動作設定
以下の機能はユーザーが設定ファイル等からON/OFFを切り替えられる設計とする。
- **ブラウザ自動表示**: アップロード成功時、取得したURLを既定のブラウザで自動的に開くか。
- **ローカルファイルの削除**: アップロード完了後、PC内の元の画像・動画ファイルを削除するか。

### 3.2. 動画（MP4）ファイルの取り扱い
Snipping Toolで録画された動画（.mp4）もアップロード対象とする。
- **アップロード形式**: 変換処理は行わず、.mp4のままサーバーへ送信する。
- **閲覧時の挙動**: サーバーは直リンクではなく、**OGPタグ（Open Graph Protocol）を埋め込んだHTMLページ**をレスポンスとして返す。
- **目的**: 取得したURLをDiscordやSlack等のツールに貼り付けた際、自動展開されてGIF画像のようにループ再生させるため。

### 3.3. アップロードファイルの命名規則
- **ハッシュ化**: ファイル名の重複防止と元ファイル名の隠蔽のため、アップロードされた**ファイル内容そのもののハッシュ値（SHA-256等）**をファイル名として採用する（URL長は短縮せず、完全なハッシュ文字列を利用する）。

### 3.4. 手動アップロード機能（拡張）
Snipping Toolの監視に加えて、以下のアップロードトリガーを実装する。
- **「送る(Send To)」メニューからの連携**: 
  - エクスプローラーで画像を右クリックし「送る」からアップロード可能にする。
  - **初回起動時のUX**: 勝手に登録せず、初回起動時に「『送る』メニューに追加しますか？」という確認ダイアログを表示する。
- **タスクトレイメニューからの連携**:
  - トレイアイコンの右クリックメニューに「ファイルを選択してアップロード」「クリップボードの画像をアップロード」機能を追加する。

### 3.5. アンインストール（お掃除）機能
レジストリを使用しない軽量アプリだが、不要になった際にSendToのショートカットや設定ファイル等を一括で削除するためのクリーンアップバッチ (`uninstall.bat`) を同梱する。

## 4. API通信仕様

### エンドポイント
- **URL**: `POST /api/upload.php`
- **Content-Type**: `multipart/form-data`

### リクエスト (Request)
| フィールド名 | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| `image` | File | Yes | アップロードするファイル本体 (png, jpg, mp4 等) |

### レスポンス (Response)
**成功時 (HTTP 200 OK)**
```json
{
  "success": true,
  "url": "https://yourdomain.com/u/8d969eef6ecad3c29a3a629280e... (ハッシュ値)"
}
```
※ 返却される `url` は、直リンクではなくOGPタグが設定されたビューアーHTMLへのリンクとなる。
