# JustLinkIt 拡張ロードマップ

これまでの開発フェーズ（Phase 1〜9）に続き、セキュリティ向上および利便性強化のための追加実装ロードマップです。
優先度（セキュリティ・安定性 > 利便性 > UI強化）順に並べています。

## 優先度：高（セキュリティ・信頼性の向上）

### 1. アップロードAPIの認証機能（APIキー実装）
- **対象**: Server / Client
- **目的**: エンドポイント（`/api/upload.php`）の不正利用を防ぐ。
- **実装内容**:
  - **Server**: `config.php` にシークレットキー（APIキー）の設定項目を追加。`upload.php` でリクエストヘッダー（例: `Authorization: Bearer <Key>`）を検証する処理を追加。
  - **Client**: `settings.json` および `AppSettings.cs` にAPIキーの設定項目を追加。トレイメニューから入力できるUIを設け、`ApiClient.cs` でPOST時にヘッダーを付与するように改修する。

### 2. 永続的なログファイルの実装
- **対象**: Client
- **目的**: ファイル監視漏れや通信エラーなどのトラブルシューティングを容易にする。
- **実装内容**:
  - アプリケーション実行フォルダ、または `%LOCALAPPDATA%` 配下に `app.log` などを出力するシンプルなロガーを実装。
  - 例外発生時や、ファイル検知・アップロード成否のタイミングでログを追記する。ファイルサイズ肥大化を防ぐため、日別ローテーションなどの簡易な仕組みも検討する。

## 優先度：中（クライアントUXと安全性の向上）

### 3. Windows起動時の自動起動（スタートアップ登録）
- **対象**: Client
- **目的**: PC起動時に自動で常駐させ、スクショ等のアップロード漏れを防ぐ。
- **実装内容**:
  - `AppSettings.cs` に自動起動のフラグを追加し、トレイの右クリックメニューでON/OFFを切り替えられるようにする。
  - ONにした際、Windowsのスタートアップフォルダに自身のショートカット（`.lnk`）を作成（または削除）する処理を実装。既存の `SendToRegistrar.cs` (WScript.Shell) のロジックを応用する。
  `uninstall.bat`も合わせて修正する。

### 4. ファイル削除機能の安全策（ゴミ箱への移動）
- **対象**: Client
- **目的**: 自動検知時の「アップロード後にローカルファイルを削除する」設定において、完全削除による意図しないデータロストを防ぐ。
- **実装内容**:
  - 既存の `File.Delete(path)` 処理を、Microsoft.VisualBasic.FileIO などを利用して「ゴミ箱へ移動（RecycleOption.SendToRecycleBin）」する処理に置き換える。

## 優先度：低（機能追加・UI拡張）

### 5. トレイメニューに「ギャラリーを開く」を追加
- **対象**: Client
- **目的**: クライアントアプリから直接ブラウザでギャラリーにアクセスできるようにする。
- **実装内容**:
  - トレイの右クリックメニュー（`TrayIcon.xaml`）に「ギャラリーを開く」メニューを追加。
  - クリック時、設定されているサーバーURLを基にギャラリーのURL（例: `https://.../gallery/`）を構築し、既定のブラウザで開く処理を実装。

### 6. ビューアー内にダウンロードボタンを追加
- **対象**: Server (Gallery Frontend)
- **目的**: ギャラリー閲覧時に、オリジナルファイルを簡単にローカルへ保存できるようにする。
- **実装内容**:
  - `public/gallery/index.html` および `app.js` のモーダルビューアー内（タグ表示部やナビゲーション周辺）に、ダウンロードアイコン（`download` 属性付きの `<a>` タグ）を配置。
  - 対象画像/動画の直リンク (`file_url`) をセットし、クリックで元ファイルがダウンロードされるようにスタイルとロジックを調整する。
